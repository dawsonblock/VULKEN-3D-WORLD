# Chunk Cache

The world streamer maintains an in-memory cache of generated chunks.  The
cache uses a **least‑recently‑used** policy and evicts old entries once the
configured capacity is exceeded.  Each entry represents one `ChunkData`
loaded from disk or procedurally generated by the world generator and fed to
the meshing pipeline.

## Tuning

`ChunkCache` exposes simple counters:

- `hits` – requests served from the cache
- `misses` – chunks that had to be generated or loaded
- `evictions` – chunks discarded to make room for new data

These metrics can be queried through `ChunkStreamer::GetCacheMetrics()` and
logged every frame or periodically.  When `misses` or `evictions` are high,
consider increasing the cache capacity; if `hits` dominate and memory usage
is a concern, a smaller capacity may suffice.

## Logging Example

```cpp
const auto& m = streamer.GetCacheMetrics();
std::cout << "cache hits=" << m.hits
          << " misses=" << m.misses
          << " evictions=" << m.evictions << '\n';
```

Use this logging while moving the camera around the world to observe how many
chunks are reused versus regenerated.  Adjust the capacity until evictions
occur infrequently while staying within memory limits.
